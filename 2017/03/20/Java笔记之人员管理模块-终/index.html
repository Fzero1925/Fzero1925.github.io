Java笔记之人员管理模块(终)

发表于 2017-03-20

Why is everything!!!

Talk is cheap, Show me the code.

实践证明这是多么的正确，最开始想着要写一个真正的项目，一个系统，但是在基本完成后才明白，自己写的东西只是很小很小的一个小模块，不用说大神，就是有点经验的人半天就写出来我两星期写的东西

但是没关系，一回生、二回熟，都是这样一步步过来的，所以自己也想坚持把这个模块完成，我也明白这个轮子肯定造了无数个了

但是当看到一个真正的项目的，一个真正的后台管理系统的时候，还是很失落，自己根本完不成一个项目，不过这种失落也是在完成了这个模块后的感觉，自己完成了一个模块的功能，再看看成熟的项目，顿时明白了差距有多么的大，短时间内很难完成，但是目前又没工作，很着急，但是面试都没通过

现在的感受以前不是没有预想过，虽然说培训班在外名声不好，但是最起码会自己完成一个项目，其实在培训班学的好的出来找工作也很容易，一切都是实力说话的，只要能通过技术面试，能完成工作，出身不重要

但是我觉得

自己身为一个计算机系的学生，如果自学都找不到工作的话，那就不要在这个行业了

这个只是我自己的固执，现在培训班的本专业的学生并不少，但是无论在哪都是要靠自己

我并不后悔毕业之后先在别的行业干了四个月，如果不是自己的固执，我本来打算在那个行业一辈子，那四个月认识了很好的同事，半师半友

在写上上一篇文章，也就是两星期前，这个模块的基本功能也都完成了，权限控制也实现了，“增删改查”里就差一个“改”的功能没有实现，因为不想和新增功能做的类似，想做成一个点击某行的修改按钮的时候，那一行显示的内容就都换成输入框和选择框，但是目前还是想不到实现的方法，这个轮子肯定有，只是自己不知道

为了实现这个功能，再加上有几个面试，花费了将近一个星期，知道我看见了一个叫jQuery EasyUI的东西

这就回到上面说的那种感受了，在完成一个重复造轮子的过程之后看见一个更好的轮子，感觉要达到那个水平好难，所以就不想再写这个项目了，通过这个重复造轮子的过程学到了很多东西，完成与不完成就不那么重要了

但是自己又不能光看书或者视频，必须有项目，基本上我是一个项目驱动者，有项目、有需求才能静下心来看东西，所以一直想到现在

写这些基本是为了自己的强迫症，因为不写个终章感觉这个系列还没完，自己又不想落下什么，有始有终嘛，虽然模块没有全完成，但是也算达到自己要的结果了，顺带的记录一下，因为这时候的心情，以后应该无法理解或想象不到，正如毕业快一年的我有点无法理解去年为什么那么的迷茫，当时是真的迷茫，现在在家时间长了，就真的有点懒了
